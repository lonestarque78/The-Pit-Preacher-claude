REMAINING ISSUES
------------------------------------------------------------

1. /api/usage/route.ts — Auth never works (CRITICAL)
   File: apps/web/app/api/usage/route.ts lines 8-16

   The service role client has no session — getUser() will always return null.
   This route will always return 401. It needs to extract the token from the
   Authorization header the same way the billing routes do, or use
   createServerClient with cookies. The same pattern was fixed in the preacher
   route — this one was missed.

2. /api/usage/route.ts — Unchecked feature param (Minor)
   File: apps/web/app/api/usage/route.ts line 21

   checkUsage(user.id, feature!) — the non-null assertion on an unvalidated
   query param will produce a confusing error if feature is omitted. Needs a
   null check and a 400 response.

3. Free tier cook limit is client-side only (Medium)
   File: apps/web/app/page.tsx lines 363-378

   The 2-cook monthly limit is checked in the browser before calling Supabase.
   There is no server-side enforcement — a user who inspects the network can
   POST directly to create a cook and bypass it. The count + insert is also a
   race condition (two tabs could both pass the check simultaneously). This
   guard needs to live in an API route or a Postgres function that does the
   count and insert atomically.

4. Two isPremium implementations that diverge (Medium)
   - apps/web/lib/billing/isPremium.ts — checks status === "active" | "trialing", ignores tier
   - apps/web/lib/premium.ts — checks tier rank, not status

   If a subscription is status: "active" but tier: "free" (edge case during
   webhook race), these return different results. Consolidate into one and
   delete the other.

5. Image base64 has no size limit (Medium)
   File: apps/web/app/api/preacher/route.ts line 53

   imageBase64: z.string().nullish() accepts any size. A large image will cause
   Anthropic timeouts and memory pressure on the server. Add .max(2_000_000)
   (~1.5MB).

6. Internal errors exposed to users (Minor)
   File: apps/web/app/page.tsx lines 402-404

   "Failed to save prep session: " + sessionError.message — raw Supabase error
   messages shouldn't reach the UI. Replace with a generic user-facing message.

7. Annual pricing wired in webhook but not in pricing page (Medium)
   The webhook's getTierFromPriceId maps four annual price IDs. The premium
   page only shows monthly prices and no toggle. If you're already paying for
   these Stripe products, this is lost conversion.


ENHANCEMENT IDEAS FOR REVENUE
------------------------------------------------------------
Ranked by conversion impact:

HIGH IMPACT:

1. Annual plan toggle on pricing page
   "Save 20%" banner with a monthly/annual switch. Annual plans dramatically
   reduce churn. The webhook already handles them; it just needs the UI.

2. Free trial for Pitmaster tier
   A 7-day trial (with card capture) converts far better than free-tier
   friction. Add trial_period_days: 7 to the Stripe checkout call. Users
   who cook once during a trial rarely cancel.

3. Cook share cards
   A /cook/[id]/share page that renders a beautiful stylized summary of a
   completed cook (what was smoked, time, temps, the Preacher's word). Users
   share these on Instagram/Facebook — it's the best free marketing you can
   get. Make sharing free, but logging and history are premium.

MEDIUM IMPACT:

4. Push notifications
   "Your brisket wrap window opens in 45 minutes." On mobile (Capacitor),
   this is achievable with local notifications tied to the cook's timeline.
   This is the killer feature for premium retention — users need to come
   back to the app.

5. Cook milestones / streak
   "You've completed your 10th brisket." Small gamification drives return
   visits. A "Pit Streak" (consecutive weeks with a logged cook) shown on
   the dashboard costs almost nothing to build and keeps free users active
   until they hit a paywall.

6. Competition mode
   KCBS/MBN turn-in timeline builder as a Pitmaster exclusive. Serious comp
   cooks will pay $11.99/mo without blinking if the tool is dialed in for
   box presentation timing.

LOWER LIFT:

7. Social proof on the pricing page
   A single Preacher-voice testimonial quote directly above the pricing
   cards converts better than empty copy.

8. Referral program
   "Give a friend 1 free month, get 1 free month." Stripe supports referral
   credits. Word-of-mouth in the BBQ community is powerful.

9. Weekly email digest
   "The Sermon This Week" — a single BBQ tip from the Preacher sent every
   Friday. Keeps the brand alive between cooks, drives re-engagement to
   start a new cook.
